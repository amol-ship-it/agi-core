# agi-core: The Universal Learning Loop

**One algorithm. Pluggable domains. Compounding intelligence.**

```
WAKE:   observe → hypothesize → execute → score → store
SLEEP:  analyze solutions → extract recurring sub-programs → compress → add to library
REPEAT: library grows → search shrinks → harder problems → compounding
```

## Prerequisites

- **Python 3.10+** (tested on 3.11)
- **Git**
- **pip**

## Setup (from scratch)

```bash
# 1. Clone this repository
git clone https://github.com/vibhor-77/agi-core.git
cd agi-core

# 2. (Recommended) Create a virtual environment
python3 -m venv .venv
source .venv/bin/activate   # Linux/macOS
# .venv\Scripts\activate    # Windows

# 3. Install dependencies
pip install -r requirements.txt

# 4. Clone the ARC-AGI dataset (needed for full benchmarks)
git clone https://github.com/fchollet/ARC-AGI.git data/ARC-AGI

# 5. Verify everything works
python -m pytest tests/ -v
```

## Syncing to latest (existing clone)

```bash
cd agi-core
git pull origin main
pip install -r requirements.txt   # in case deps changed

# If you already cloned ARC-AGI, update it too:
cd data/ARC-AGI && git pull && cd ../..
```

## Running experiments

```bash
# Run tests first (always)
python -m pytest tests/ -v

# Run the ARC-AGI Phase 1 experiment on built-in sample tasks (no dataset needed)
python -m experiments.phase1_arc

# Run on ARC-AGI-1 training data (requires step 4 above)
python -m experiments.phase1_arc \
  --data-dir data/ARC-AGI/data/training \
  --rounds 5 \
  --beam-width 200 \
  --max-generations 60

# Run on a subset of tasks (faster iteration)
python -m experiments.phase1_arc \
  --data-dir data/ARC-AGI/data/training \
  --max-tasks 50 \
  --rounds 3

# Run the symbolic regression demo (simplest domain, no dataset needed)
python -m grammars.symbolic_math

# Run ARC sample tasks directly (no dataset needed)
python -m grammars.arc

# Run tests with coverage
python -m pytest tests/ -v --cov=core --cov=grammars --cov-report=term-missing
```

Results are saved to `experiments/results/`.

## What the Phase 1 experiment does

1. Loads ARC-AGI tasks (or built-in samples if no dataset)
2. Sorts by difficulty (curriculum order)
3. Runs wake-sleep rounds: solve → extract library → solve harder tasks
4. Prints the **compounding curve** — the key metric
5. Saves learned library to `experiments/results/`

## The 4 interfaces

Every domain implements exactly 4 things:

| Interface | What it does | Symbolic Math | ARC-AGI | Zork |
|-----------|-------------|---------------|---------|------|
| **Environment** | Execute programs, return results | Evaluate formula on x | Apply grid transform | Execute game action |
| **Grammar** | Define primitives, compose/mutate | sin, cos, +, ×, ... | rotate, flip, crop, ... | move, take, use, ... |
| **DriveSignal** | Score: how wrong + how complex | MSE + node count | Pixel edit distance + size | Game score + novelty |
| **Memory** | Store episodes, library, solutions | InMemoryStore | InMemoryStore | InMemoryStore |

The core loop (`core/learner.py`) depends **only** on these interfaces. It never imports anything domain-specific.

## The key metric: the compounding curve

```
Round  Solved     Rate  Library  New  Avg Energy   Wake(s)  Sleep(s)
---------------------------------------------------------------------
    1    2/4     50.0%        3    3      0.0012       4.2       0.1
    2    3/4     75.0%        5    2      0.0008       3.8       0.1
    3    3/4     75.0%        6    1      0.0005       3.1       0.0
```

If solve rate increases across rounds without new hand-coded primitives, the framework is working.
